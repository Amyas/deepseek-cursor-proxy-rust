# 行为对齐基线

本文档用于约束 Rust 版本与 Python 版本在核心协议行为上的一致性。

## 第一批必须对齐的行为

- 请求路径只接受 `/chat/completions` 和 `/v1/chat/completions`
- 需要 Bearer Token
- `functions` / `function_call` 兼容转换为 `tools` / `tool_choice`
- `reasoning_effort` 的别名归一化
- assistant 历史中的 thinking 展示块在发往上游前必须剥离
- 缺失 `reasoning_content` 时支持 `recover` / `reject` 策略
- 非流式响应中的 `reasoning_content` 可提取并入库
- 流式响应中的 `reasoning_content` 和 `tool_calls` 可增量累积

## 第一阶段不要求完成的行为

- 完整 HTTP 代理
- 完整 SQLite 缓存实现
- 完整 SSE 解析
- trace 文件落盘
